# TGLabs Content Workflow — Writer Agent

- **Website:** https://www.tglabs.info
- **Platform:** เกมมือถือ (ไทย)

---

## Workflow: Daily News Pipeline

### ขั้นตอนที่ 1 — หาข่าว
1. ใช้ `web_search` ค้นหาข่าวเกมมือถือวันนี้
2. เลือกข่าวที่น่าสนใจ จากแหล่งเช่น:
   - IGN, GameSpot, Polygon
   - Rappler, Inquirer (เอเชียตะวันออกเฉียงใต้)
   - เว็บไซต์ข่าวเกมไทย
3. ตรวจสอบว่ายังไม่เคยลงข่าวนี้แล้ว (เช็ค Supabase หรือเว็บ)

### ขั้นตอนที่ 2 — สรุปและตรวจสอบ
1. อ่านข่าวจาก URL ต้นทาง
2. ตรวจสอบความถูกต้อง:
   - วันที่อัปเดต
   - แหล่งที่มาน่าเชื่อถือ
   - ข้อเท็จจริงหลัก (version, patch, ชื่อตัวละคร, ฯลฯ)
3. หาข้อมูลเพิ่มเติมจากแหล่งอื่นถ้าจำเป็น (cross-check)

### ขั้นตอนที่ 3 — เขียนข่าว
ใช้โครงสร้างและกฎจาก **TGLabs_Content_Guidelines.md**

### ขั้นตอนที่ 4 — ส่ง Preview
ส่งข้อความ Discord ให้ **พี่แบงค์ (TopGameTH)** ตรวจ Approve ก่อนลงข่าว

```
📰 [รอ Approve] <title>
🔗 <source_url>
📝 Excerpt: <excerpt>
---
<content preview สั้นๆ 2-3 บรรทัด>
---
✅ Approve ลงข่าว
❌ ขอแก้ไข / ข้อเสนอแนะ
```

### ขั้นตอนที่ 5 — ลงข่าว (เมื่อได้รับ Approve)
1. บันทึกลง Supabase ผ่าน API ที่กำหนด
2. ตั้ง `status: "publish"` หรือ `status: "draft"` (ตามที่พี่แบงค์สั่ง)
3. Report ผลกลับ

---

## การ Approve ผ่าน Discord
- ส่ง preview ไปที่ช่องที่ตกลงกัน
- รอ React ✅ หรือข้อความยืนยันจากพี่แบงค์
- ถ้าขอแก้ไข → แก้ตาม feedback แล้วส่งใหม่

---

## ความถี่
- **ทุกเช้า 08:00 น. (เวลาไทย)** ผ่าน Cron Job
- ถ้าวันนั้นไม่มีข่าวน่าสนใจ → รายงาน "วันนี้ไม่มีข่าวที่เหมาะสม"

---

## หมายเหตุ
- ถ้าไม่แน่ใจเรื่องข้อเท็จจริง → ตรวจสอบจากแหล่งทางการของเกมนั้นๆ ก่อนเสมอ
- ถ้าข่าวมีประเด็นละเอียดอ่อน (เช่น ban, scam, ปัญหาทางกฎหมาย) → รายงานพี่แบงค์ก่อนลง
